# Release Policy

Use Semantic Versioning where applicable.

```text
MAJOR.MINOR.PATCH
```

- **MAJOR**: breaking changes, incompatible API changes, major logic or architecture changes.
- **MINOR**: backward-compatible features, enhancements, UI improvements, new optional capabilities.
- **PATCH**: bug fixes, small documentation fixes, safe maintenance updates.

Version numbers are compatibility contracts, not decoration.

Every release should ideally include summary, added, changed, fixed, deprecated, removed, breaking changes, and migration notes when needed.
